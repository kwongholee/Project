-다른 페이지들 만들어지면 옆에 그리로 이동할 수 있는 링크 걸어놓으면 좋을 듯

소개페이지('/introduce')
-내가 이 페이지들을 만들게 된 이유 서사(포트폴리오 용도, 경험 등등)
-애플 공홈에서 사용되는 sticky 기능 사용하면 좋은 ui 만들 수 있을 듯
-개발자스러운 소개글 만들기(배경사진으로 내가 작성한 코드를 보여준다든지)

개인프로필페이지('/profile/:id')
-개인 프로필 사진, 이름, 이메일, 닉네임, 전화번호, pw, 소개글 보여주기
-반드시 라우터에 URL parameter로 id 표현할 것(라우터 구분 위해서)
-개인 페이지는 이후 글쓴이의 상세정보 페이지로 보여질 수 있음
-원하는 정보만 공개 가능하도록 설정창 만들기

글쓰기페이지('/write')
-글 쓰기 가능(추후에 사진, url, 동영상, 코드블럭 등등 넣기)
-임시 저장 기능을 구현할까? (그럴려면 db 공간을 좀 마니 차지하긴 함 위에 글 말고 다른 자료들도 올릴 거 생각하면 그럼 일단 대기)
=> atlas 말고 실제 몽고디비 사용해도 되긴 하는데 그건 좀 번거로울지도? 너무 오랜만에 만지는 건데(그럼 일단 대기!)
-글쓴이를 익명으로 대체 가능(이때 디비에는 익명 아이디 안 겹치는 선에서 무작위 랜덤생성해야할 듯)
-글 제목을 오직 한글만 되게 할 것인가에 대해서는 고민해봐야할 듯(because of 몽고디비 정렬)
-글자수 제한 두기

로그인페이지('/login')
-id랑 pw 입력하면 '/'로 이동
-밑에 회원가입페이지로 이동할 수 있는 링크 걸기
-OAuth 2.0 기능 추가 가능?

회원가입페이지('/signup')
-개인 프로필 사진, 이름, 이메일, 닉네임, 전화번호, pw, 소개글 적기
-정보 이용 동의 만들기
-제출 버튼 누르면 '/personalpage:id'로 이동

게시판페이지('/list')
-정렬 기능 사용 가능(좋아요순, 작성 시간 순, 등등) - 일반적으로 첨 들어왔을 때는 작성 시간순으로 배열되어 있음
-게시글이 보이는 갯수는 5개 or 10개
-하단 중앙에 검색창 기능
-게시글 제목을 누르면 해당 게시글 상세페이지로 들어감

게시물 페이지('/feed/:_id')
-해당 게시물이 가지고 있는 고유 아이디를 url parameter로 사용할 것(라우터 구분 위해)
-게시물 상단 또는 하단에 좋아요 버튼 생성(누른 횟수에 따라 취소 가능)
-댓글 + 대댓글 기능(이거 추가해도 디비 안 터지겠지?)
-본인이 작성한 게시물이면 수정이나 삭제 기능 추가
